# Flow Primitives

Sprint stage 文件中使用的流程控制原语。每个原语语义单一，正交不重叠。

## 流程控制（6 个）

### STOP — 停下等人

**语法**: `[STOP]` / `[STOP:confirm]` / `[STOP:choose]` / `[STOP:respond]`

| 参数 | 交互模式 | 用户要做什么 |
|------|---------|------------|
| `confirm`（默认） | 确认放行 | 只认白名单词：ok / yes / continue / 确认 / 同意 / 好 / 对 / 可以 |
| `choose` | 从选项中选 | 正文列出选项，用户选一个 |
| `respond` | 给出实质内容 | 用户给出回答/反馈后才继续 |

规则：
- 输出内容后停下等用户回复
- `confirm`: 非白名单词 = 还在沟通，不是放行
- `choose`: 用户必须选一个列出的选项
- `respond`: 用户给出实质回复（不是"嗯""好"）后才进下一步

### INFO — 输出不停

**语法**: `[INFO]` / `[INFO:warn]`

| 参数 | 语义 |
|------|------|
| （默认） | 中性状态通知 |
| `warn` | 警告，需要注意但不阻塞，输出前缀 `[警告]` |

规则：必须有输出模板，输出后直接继续。

### BRANCH — 按条件选路

**语法**: `[BRANCH]`

规则：
- 必须附条件-路径表
- 条件必须是可观测的值（脚本返回码、文件是否存在、数值比较、参数值）
- 不能用 LLM 主观判断作为条件
- 路径可以引用其他原语（如 `→ [STOP]`、`→ [RETRY:1]`）

### SKIP — 不执行

**语法**: `[SKIP: 条件]` / `[SKIP:ask]`

| 参数 | 语义 |
|------|------|
| 条件 | 条件满足自动跳过 |
| `ask` | 建议跳过，问用户确认 |

规则：
- 跳过时必须输出：`[跳过] {单元名} — {原因}`
- 不能默默跳过
- `ask` 时用户说 ok 才跳，说不跳则正常执行

### RETRY — 重做同一个

**语法**: `[RETRY:N]`（N ≤ 2）

规则：
- 重试前输出：`[重试] {单元名} ({当前次数}/{N})`
- 超过 N 次自动升级为 `[STOP]` 等用户介入
- 重试范围限定在当前 step 内

### BACK — 回到之前的

**语法**: `[BACK: 目标]`

规则：
- 目标必须是显式的 step 或 stage 名，不能是"上一步"
- 回退前输出：`[回退] {原因}，回到 {目标}`
- 回退不计入重试次数

## 动作标记（5 个）

### BASH — 必须用 Bash 执行

**语法**: `[BASH]`

规则：标记的命令必须用 Bash 工具真正执行，不是口头描述。

### TASK — 创建跟踪任务

**语法**: `[TASK: 标题]`

规则：看到标记时用 TaskCreate 工具创建任务，完成时用 TaskUpdate 标记 completed。

### CHECKLIST — 逐条验证

**语法**: `[CHECKLIST]`

规则：
- 正文中每条 `- [ ]` 必须逐条验证
- 不能跳条，不能批量说"全部通过"
- 每条验证后标注：`[x]` 通过 / `[!]` 失败
- 任一条失败则整体失败

### PROGRESS — 追踪进度

**语法**: `[PROGRESS]`

规则：标记的循环/批量操作需要显示进度，每完成一个单元更新。

### WORKTREE — 隔离工作区

**语法**: `[WORKTREE]`

规则：看到标记时用 EnterWorktree 创建隔离工作区，完成后 ExitWorktree 退出。

## 组合规则

1. 一个 step 最多一个主原语（标题上的标记）
2. BRANCH 的路径可以引用其他原语
3. RETRY 超限自动升级为 STOP
4. SKIP 优先级最高：条件满足时其他原语不生效
5. 动作标记可以和流程控制共存
